## Why a String Is Really a List of Characters

A student asked why one part of the code uses brackets with a position number and another part doesn't. This led to a deeper point: under the hood, a string like "hello" is stored as a list of single characters, even though it looks and behaves like one solid piece of text to us. That's why the same `len()` works on both lists and strings.

## Two Ways to Loop

We compared two styles of loop: the traditional version, which counts through positions one at a time, and a simpler version that goes "for every word in this list" without tracking a position number. The simpler version is easier to read for going through everything in order, but the traditional version is still needed for trickier jobs, like grabbing only every other word.

## Plain vs. Enhanced For-Loop, Side by Side

Same list of strings, two ways to visit every item.

**Plain for-loop** — counts through positions with an index variable, then uses that index to reach into the list:

```python
words = ["relentless", "summer", "Chicago"]
for i in range(len(words)):
    print(words[i])
```

Here `i` takes the values `0`, `1`, `2`, and `words[i]` is what does the actual work of fetching each word. Because we have the index `i` on hand, we can also do position-based tricks, like `range(0, len(words), 2)` to grab only every other word.

**Enhanced for-loop** — skips the index entirely and lets the loop variable be each item in turn:

```python
words = ["relentless", "summer", "Chicago"]
for word in words:
    print(word)
```

There is no `i`, no `len()`, and no `words[i]` — `word` *is* the current string on each pass. This reads closer to plain English ("for every word in words") and is the right choice whenever the job is simply "do something with each item," with no need to know *where* in the list that item lives.
