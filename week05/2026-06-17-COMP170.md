## Why Lists Matter

We picked back up with our sentence-splitting example from yesterday: take a sentence and find every word longer than a given length. The big idea today was that once you turn a sentence into a list of words, most of the hard work is already done — you just walk through the list and check each word. As a fun detour, we talked about where artificial intelligence actually got its start (a janitor at the University of Chicago in 1941!) and how AI only became practical once computers had enough memory to work with large lists of numbers.

## Packaging Code into a Method

Instead of leaving our word-checking code loose, we wrapped it in a method called `filter_words`, which takes a sentence and a cutoff length and gives back a list of the matching words in capital letters. We added type hints (notes like "this should be a string") — Python doesn't enforce these, but they tell other programmers what we intended. We also wrote a docstring, a block of text right under the method's first line that explains what the method does, what it expects, and what it gives back.

## Checking the Input Before Using It

Before processing anything, we added checks: the sentence shouldn't be empty, and the cutoff length should be a reasonable number (not negative, not absurdly large). We combined three separate `if` checks into one using `and`, since all three conditions need to be true at the same time. We told the story of a real airline web form that didn't check ages, so thousands of users typed "11111" just to finish the form — a good reminder of why checking input matters.

## Running the Code

Finally, we saw how to actually run our method from another file by importing it, and as a shortcut for class demos, by adding an `if __name__ == "__main__":` block at the bottom of the same file.
