# Naming Variables in Loops

We followed up on this week's assignment, where Bella asked about the loop that checks whether each word in a list is made only of letters and numbers, or only of letters. We showed that the name we choose for the loop variable doesn't change what the code does — `for word in test_strings`, `for broccoli in test_strings`, or even `for x in test_strings` all work exactly the same way, as long as the same name is used consistently inside the loop. What matters is choosing a name that helps a reader understand what each item in the list represents.

## Adding Up a List: the Accumulator Pattern

We built a small program that computes the average of a list of daily temperatures. To add up all the values, we first set a variable called `sum` to zero before the loop starts, then added each temperature to it one at a time as the loop ran. We explained why that starting value matters: the computer cannot guess that an "empty" variable should be treated as zero, so we have to say so explicitly. This pattern of building up a running total by repeating the same step for every item in a list works the same way for multiplying values together, not just adding them.

## Factorials, the Slow Way and the Smart Way

We used multiplication to compute factorials (4 factorial is 1x2x3x4, which is 24), first with a simple loop, then rewrote it so the method calls itself on a smaller number each time, stopping at the simplest case, 0 factorial, which is defined to equal 1. This self-calling approach is called recursion, and we'll build on it as we start writing our own methods for working with words and lists next class.
