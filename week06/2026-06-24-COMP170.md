## Fixing the Multi-Space Bug in `our_split`

We picked up from yesterday's `our_split` method, which breaks a sentence into words by scanning character by character. The method worked when words were separated by exactly one space, but produced unwanted empty strings when two or more spaces appeared between words. We traced the logic together: every time we encounter a space, whatever is in the temporary "notepad" (the variable `new_entry`) gets added to the output and the notepad resets. If the next character is also a space, an empty notepad gets added again — and again. The fix was a single `if` statement: only append `new_entry` to the output when `len(new_entry) > 0`. One conditional, problem solved.

## What a Method Header Means

With Erica's question as a starting point, we broke down the `def` line: the method's name, what information it needs (the sentence and the delimiter), and what it gives back (a list of words). The type hints — `sentence: str`, `delimiter: str = ' '` — don't change how the code runs; they communicate intent to anyone reading the code later, including ourselves. The default value `' '` means that if no delimiter is provided, the method assumes a space.

## Methods as a Way to Manage Complexity

We spent time on the deeper idea: a method is a self-contained step in solving a larger problem. Getting a degree means splitting 120 credits into 40 manageable courses. Driving to Denver means planning an overnight stop in Nebraska. Programming works the same way — one goal per method, one method per goal. The most important skill isn't Python syntax; it's the ability to express a solution as a sequence of small, clear steps.
