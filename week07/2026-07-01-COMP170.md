# July 1 — COMP 170

## Counting How Many Times a Letter Appears

We moved from asking "does this letter appear?" to "how many times does it appear?" With Rebecca's help, we wrote the plain-English plan first: look at every letter one at a time, and every time it matches the one we're searching for, add one to a running total. We tried this three ways — a loop that goes through each position by number, a loop that visits each letter directly, and a compact one-line version — and talked about why the simplest, most old-fashioned version is often the best choice: code that anyone, in any language, could read and understand.

## Trying the Prompt on Claude

We tried giving Rebecca's plain-English plan directly to Claude as a prompt, and it produced the same solution we'd just written by hand — a good reminder that a clear, specific prompt matters as much as the tool you give it to.

## Finding and Fixing Bugs

We tested our counting code and found two mistakes. First, running the count twice by accident doubled our result. Second, after adding a safety check for missing input, we accidentally started counting from the wrong position, which skipped a letter and gave one too few. Both were fixed by carefully re-checking where the counting actually starts — a reminder that in this class we always start counting from position zero.

## Protecting Code from Missing Input

We saw what happens when the code is given empty or missing information instead of a real sentence to search — the program crashed. We fixed this by checking first that both the sentence and the letter we're looking for actually exist before doing any work, so the method returns a clearly invalid answer (like $-1$) instead of crashing.
