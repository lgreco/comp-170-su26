## From Pseudocode to Python

We picked up from yesterday's pseudocode assignment. With Bella's help, we walked through her solution to finding the position of a character in a string — returning $-1$ if the character isn't present. This gave us a clean example to distinguish two ideas: a method's **inputs** become its parameters, and its **output** is its return value. These are easy to confuse but mean very different things when you're designing a method.

## Loops: Definite and Indefinite

We introduced **loops** as a way to repeat a sequence of steps. There are two kinds:

- **Definite loops** repeat a fixed number of times known in advance. With Erica's help, we counted from 1 to 5, then counted even numbers up to 10 — clear stopping points set from the start.
- **Indefinite loops** repeat until a condition changes. With Andrew's help, we demonstrated counting up from 0 until "I fall asleep" — unpredictable in timing, but eventually true.

Both kinds must terminate. A loop that never stops is an **infinite loop**. We saw two common causes: a stopping condition that can never become false (e.g., `while i > -10` when `i` starts at 0 and keeps increasing), and forgetting to update the loop variable so it stays at 0 forever. We also demonstrated using `%` (modulo) to print a progress dot every 100,000 steps — a practical way to confirm a long-running program is still moving and hasn't stalled.

## Boolean Logic and Loop Conditions

We connected the character-finding loop to **Boolean logic**: the `while` condition is an AND expression — keep going as long as we haven't reached the end of the string AND haven't found the target yet. As soon as either condition fails, the loop stops. We reviewed truth tables for AND and OR, and briefly introduced XOR (true only when exactly one input is true, like choosing one dish from a menu) and NAND (the inverse of AND). Understanding how these connectives combine lets us write precise loop conditions that stop exactly when we want them to.
