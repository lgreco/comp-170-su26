## What is a file?

We opened by asking what a "file" really is. With Erica, Rebecca, and Evan's answers — "a place to store things," "information in a specific place," "a file folder" — we found that everyday definitions blur together, and that a file is best understood as any container of information, no matter its physical form.

## Files are naturally hierarchical

Using the example of an old-fashioned office — pages inside manila folders, inside hanging folders, inside drawers, inside cabinets, inside rooms, inside a building — we saw that files nest inside other files, forming a hierarchy. We connected this to something even older and more familiar: a family tree, which organizes people the same nested way.

## How computers came to store files

We traced how computers moved from physically punched cards, to magnetic tape (which can only be read in order, like a cassette), to spinning magnetic disks (which allow jumping straight to any point, like a vinyl record needle). Spinning hard disks read data using a tiny head riding on a cushion of air only a hundredth of an inch above the disk — which is why a spec of dust could cause a "disk crash." Around 20 years ago, this mechanical storage was mostly replaced by electronic storage, the same technology used for a computer's working memory, but built to keep its data even when the power is off.

## The three-tier memory hierarchy

We introduced the von Neumann hierarchy: registers (tiny, extremely fast memory inside the processor), main memory (larger, still fast, but wiped when power is lost), and storage (large, slower, and permanent). Files live in storage and get pulled temporarily into main memory whenever a program needs to work with them.
