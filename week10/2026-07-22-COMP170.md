## Creating and writing to files

We explored files hands-on by splitting the screen: Python on one side, the folder contents on the other, so we could watch files appear in real time. Opening a file for writing creates it — we saw a new file (`broccoli_icecream.txt`, then a simpler `demo.txt`) show up in the folder the moment we ran the open command, even though it started out completely empty.

## Why writes don't show up right away

When we wrote a line to `demo.txt` and checked its contents, nothing appeared — twice. What we were writing was sitting in a temporary holding area called a buffer, not yet saved to the file. Only after we closed the file did our text actually show up. We confirmed the data survives after quitting and restarting Python, because it lives in storage, not in the computer's memory — memory empties out when a program ends, but storage does not.

## Two ways to write, one way to read

Opening a file in write mode wipes out anything already in it and starts fresh; opening in append mode keeps what's there and adds new lines to the end. We compared this to a waiter using a fresh notepad page (write) versus adding more items to an existing order (append). Opening a file in read mode lets us look at its contents without any risk of changing them — trying to write to a file opened this way simply does nothing.

## Reading a file line by line

We wrote a short program that opens a file and reads it one line at a time until there's nothing left. We noticed `print()` adds its own line break after whatever we tell it to print, so if our text already ends in a line break, we get an extra blank line — something we can fix with the `.rstrip()` string command, which trims characters off the right end of a string.

## Files, strings, and the bigger picture

We closed by tying files back to the "four pillars" of computing: words made of letters and symbols (strings), repeating instructions (loops), yes/no decisions (if statements), and lists of items (arrays). Even the keyboard is treated as a file that the computer keeps reading from as we type. We traced a short history from old comma-separated student records — where adding one new detail like a middle name could break the whole layout — to labeled formats like XML, JSON, and YAML that describe what each piece of data means.
