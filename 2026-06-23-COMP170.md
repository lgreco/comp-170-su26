## Reinventing the Split Method

Today's goal was to build our own version of the `split` method — the tool we already use to break a sentence into a list of words — without using it. We started with a sentence written out one letter per box on a sheet of squared paper, with the rule that we could only look at one box at a time, and that anything we needed to remember had to be written down rather than kept in our heads.

## Building the Steps Together

With help from several of us, we worked out the process box by box: number the boxes starting at 0; look at the current box; check whether it holds a letter or a space; if it's a letter, copy it to the output and move to the next box; if it's a space, start a new line in the output and move to the next box; keep repeating until there are no boxes left. This step-by-step plain-language description is called pseudocode — instructions written one per line, with no fixed rules for how to format it, but with the goal of being clear and simple enough to translate directly into code.

## Turning Pseudocode into Python

We translated the pseudocode into a method called `our_split`, using a position counter to read one character at a time from the sentence, copying letters into a growing word and starting a new word whenever we hit a space. Along the way we found two real bugs: the running word needed to start outside the loop, and the very last word needed to be added after the loop ends, since nothing tells the code to add it otherwise. We also saw that this same approach can split on any character, not just spaces. Homework: write pseudocode for more "reinvented" string methods.
